# `market-survey-whitepaper` — TEMPLATES.md

## Purpose

Use these templates to build a **market-facing, gated, PR-supporting survey results whitepaper** for Health Shield.

Each template is designed to help you produce sections that are:

- clear and easy to scan
- evidence-led and journalist-friendly
- warm, credible, and useful
- suitable for gated publication
- ready for PR support and wider campaign use
- aligned to Health Shield’s compliance and brand guardrails

Use these templates alongside:

- `SKILL.md`
- `RUBRIC.md`
- `EXAMPLES.md`
- `/shared/style.md`
- `/shared/voice.md`
- `/shared/accessibility.md`
- `/shared/legal-safety.md`

---

## Core rules for all templates

Before drafting any section, follow these rules:

- Frame findings as **survey findings**
- Do **not** describe findings as nationally representative
- Do **not** present survey findings as universal fact
- Do **not** include competitor references
- Keep product mentions **soft and non-promotional**
- Avoid legal, medical, or regulated-advice tone
- Prefer clear, quotable, evidence-led phrasing
- Avoid sensational or alarmist wording
- Make key findings easy for journalists to lift accurately
- Include the required regulatory wording **exactly as written** in the final asset

## 1. Cover/title block

### Purpose

Introduce the report clearly and give readers an immediate sense of:

- the topic
- the audience
- the year or reporting period
- the main angle of the research

This section should help the asset feel valuable enough to download and easy to understand at a glance.

### When to use

Use for every survey results whitepaper.

### Required inputs

| Input            | Notes                                                              |
| ---------------- | ------------------------------------------------------------------ |
| Report title     | Clear, specific, campaign-aligned                                  |
| Subtitle         | Optional but useful for context                                    |
| Publication year | Usually included in title or subtitle                              |
| Audience angle   | For example: employer view, employee view, broker view             |
| Brand name       | Health Shield or Health Shield Friendly Society, where appropriate |

### Writing guidance

- Keep the main title short and clear
- Make the subtitle do the harder explanatory work
- Focus on the insight topic, not the product
- Keep brand presence measured
- Avoid sales-led phrasing
- Make the wording feel credible and useful
- Use wording that would still make sense if quoted in media coverage

### Markdown skeleton

```
# [Report title]

## [Subtitle or explanatory line]

**Published:** [Month Year]  
**By:** Health Shield
```

## 3. Executive summary

### Purpose

Summarise the biggest findings and why they matter.

This section should give busy readers, journalists, and stakeholders a fast, useful overview of the report’s value.

### When to use

Use for every whitepaper. Draft early to shape direction, then refine once the report is complete.

### Required inputs

| Input                    | Notes                                |
| ------------------------ | ------------------------------------ |
| Main story of the report | The central takeaway                 |
| Topline findings         | The strongest 3 to 5 stats or themes |
| Campaign objective       | To keep the summary focused          |
| Audience relevance       | Why the findings matter now          |
| Any year-on-year shift   | If relevant and approved             |

### Writing guidance

- Open with the main issue in plain English
- Use the strongest findings early
- Explain why the findings matter
- Keep the tone calm and evidence-led
- Write with media lift in mind
- Use bullets if that makes the section easier to scan
- Avoid overloading the summary with too many numbers
- Keep findings attributed to the survey or report
- Do not drift into recommendations that sound like formal advice

### Markdown skeleton

```
## Executive summary

[Report name] explores [main issue] and what it means for [target audience].

Our latest survey findings highlight a clear picture: [plain-English summary of the core story]. While [context or tension], the findings suggest that [what is changing / what readers need to notice].

Key findings include:

- **[Stat or finding 1]** – [Why it matters in one sentence]
- **[Stat or finding 2]** – [Why it matters in one sentence]
- **[Stat or finding 3]** – [Why it matters in one sentence]

[Optional short paragraph on year-on-year change, subgroup pattern, or wider context.]

Taken together, these findings show [measured conclusion tied to the report theme]. This report sets out the evidence in more detail and explores what today’s pressures may mean for [audience/context].
```

## 4. Introduction

### Purpose

Set up the report, explain why it exists, and give readers a warm, credible welcome into the story.

### When to use

Use near the front of every whitepaper, after the executive summary.

### Required inputs

| Input                     | Notes                                |
| ------------------------- | ------------------------------------ |
| Report purpose            | Why this report was created          |
| Current context           | Market, workforce, or sector context |
| Audience                  | Who the report is for                |
| Health Shield perspective | Light-touch and relevant             |
| Named spokesperson        | Optional                             |

### Writing guidance

- Explain the reason for the report clearly
- Ground the report in the audience’s real pressures
- Use a warm, supportive tone without sounding casual
- Keep the brand role helpful and measured
- Focus on helping the reader understand the issue
- If using a named spokesperson intro, keep it concise and relevant
- Do not turn this into a product pitch

### Markdown skeleton

```
## Introduction

[Name, role, organisation — optional]

This report is based on our latest survey of [audience/group], giving a clear view of [topic] in [year or period].

At a time when [context], it is becoming harder for [audience] to [problem or challenge]. While [surface-level positive or commonly assumed view], the findings in this report suggest there are deeper pressures shaping everyday experience.

We hope this report gives [target audience] useful evidence and practical insight to help them better understand [topic]. By looking closely at what respondents told us, we can see where the biggest pressures sit, where needs are changing, and where support may make the greatest difference.
```

## 5. Methodology section

### Purpose

Show how the research was carried out and give readers confidence in the findings.

### When to use

Use for every whitepaper.

### Required inputs

| Input                | Notes                            |
| -------------------- | -------------------------------- |
| Research agency name | If used                          |
| Commissioner         | Health Shield                    |
| Sample size          | Exact figure                     |
| Audience surveyed    | Who took part                    |
| Fieldwork dates      | Exact dates if available         |
| Standards followed   | MRS, BPC, ESOMAR, or equivalent  |
| Comparability caveat | If year-on-year data is included |

### Writing guidance

- Keep the wording clear and factual
- Include enough detail to support credibility
- Avoid overstating the scope of the findings
- If comparisons are used, add any caveats clearly
- Make sure the methodology supports proper attribution
- Keep this section neutral in tone

### Markdown skeleton

```
## Survey methodology

This report is based on quantitative research commissioned by Health Shield and carried out by [research agency].

### Survey design and fieldwork

- **Audience surveyed:** [Description]
- **Sample size:** [Number]
- **Fieldwork dates:** [Date range]
- **Location:** [Geography, if relevant]
- **Method:** [Online survey / other method]

[Research agency] is a member of [relevant bodies] and follows [relevant standards or codes], helping to support data quality and research good practice.

### Interpretation and comparability

[If relevant: Year-on-year comparisons are included to provide context. Some variation may reflect changes in sample composition, fieldwork conditions, or methodology, so comparisons should be interpreted with care.]

The findings in this report should be understood as survey findings from this research sample.
```

## 6. Year-on-year insights section

### Purpose

Highlight meaningful changes between reporting periods without overstating them.

### When to use

Use only when there is a previous survey or comparable dataset that has been approved for comparison.

### Required inputs

| Input                    | Notes                                 |
| ------------------------ | ------------------------------------- |
| Previous period findings | Approved comparison data              |
| Current period findings  | Validated current data                |
| Comparability note       | Required if methods or samples differ |
| Key trend areas          | Usually 3 to 5 high-value comparisons |

### Writing guidance

- Compare only what is valid to compare
- Focus on meaningful shifts, not every small change
- Make trend statements careful and precise
- Explain what changed and why it matters
- Add a caveat if methodology or sample changes may affect comparison
- Avoid dramatic language about change unless clearly supported

### Markdown skeleton

```
## Year-on-year insights: [Previous year] vs. [Current year]

A comparison of findings from [previous year] and [current year] offers useful context on how [topic] may be changing over time.

### [Trend area 1]

In [previous year], [finding]. In [current year], [finding].

[One or two sentences explaining what this may suggest and why it matters.]

### [Trend area 2]

In [previous year], [finding]. In [current year], [finding].

[One or two sentences explaining what this may suggest and why it matters.]

### [Trend area 3]

In [previous year], [finding]. In [current year], [finding].

[One or two sentences explaining what this may suggest and why it matters.]

### Summary of key trends

- [Trend summary 1]
- [Trend summary 2]
- [Trend summary 3]

[Optional comparability note.]
```

## 7. Main findings chapter template

### Purpose

Present one major area of the research clearly, with a strong narrative, supporting data, and explanation of why it matters.

### When to use

Use once for each main findings chapter.

### Required inputs

| Input                  | Notes                      |
| ---------------------- | -------------------------- |
| Chapter title          | Main theme of the section  |
| Chapter intro point    | What this section is about |
| Key stats              | Main evidence              |
| Subgroup findings      | Optional but often useful  |
| Context or implication | Why the finding matters    |
| Chart references       | Optional                   |
| Pull-quote-ready line  | Optional but encouraged    |

### Writing guidance

- Open with the point, not the process
- Start with the strongest evidence or clearest takeaway
- Keep paragraphs short
- Use subheadings for scanability
- Explain what the finding means in practice
- Make standout figures easy to lift into PR or media use
- Use subgroup detail to add depth, not clutter
- Keep the tone measured and useful
- Avoid turning findings into broad claims about the whole population
- Avoid hard-sell product transitions

### Markdown skeleton

```
## [Chapter title]

[Short opening paragraph explaining the theme of the chapter and why it matters.]

### [Subheading focused on a specific finding]

[Lead sentence with the key point.]

- **[Stat 1]**
- **[Stat 2]**
- **[Stat 3]**

[Short paragraph explaining what the findings suggest.]

[Optional subgroup paragraph: Among [subgroup], the findings show [detail].]

### [Second subheading]

[Lead sentence with the key point.]

[Supporting evidence in short paragraph or bullets.]

[Short interpretation paragraph focused on reader value and real-world meaning.]

### [Third subheading if needed]

[Lead sentence with the key point.]

[Supporting evidence.]

[Measured explanation.]

> [Optional pull-quote-ready line based on the evidence.]

[Optional chart reference placeholder: `Q[number]. [Question wording]`]
```

## 8. Key observations box

### Purpose

Summarise the most important takeaway from a chapter in a short, high-value format.

### When to use

Use at the end of a chapter or after a strong cluster of findings.

### Required inputs

| Input              | Notes                             |
| ------------------ | --------------------------------- |
| Main takeaway      | One key message                   |
| Supporting context | One or two lines                  |
| Evidence basis     | Must reflect the chapter findings |

### Writing guidance

- Keep this short and clear
- Write in complete sentences
- Focus on what the findings suggest
- Make it quotable, but not dramatic
- Do not overstate certainty
- Do not introduce new evidence here

### Markdown skeleton

```
### Key observation

[One short paragraph that captures the chapter’s main takeaway in a clear, evidence-led way.]

[Optional second sentence linking the takeaway to business, workforce, or market relevance.]
```

## 9. Expert commentary section

### Purpose

Add informed perspective that helps readers interpret the findings without replacing the survey evidence.

### When to use

Use when you have an approved expert contribution from:

- a Health Shield spokesperson
- an external commentator
- an industry expert
- a broker or partner voice

### Required inputs

| Input                    | Notes                          |
| ------------------------ | ------------------------------ |
| Contributor name         | Full name                      |
| Contributor title        | Job title and organisation     |
| Commentary text or brief | Approved content               |
| Relevant chapter/theme   | What the commentary relates to |
| Bio summary              | Optional but useful            |

### Writing guidance

- Keep the commentary aligned to the report findings
- Make sure the expert adds meaning, not repetition
- Use a natural editorial voice
- Keep recommendations measured
- Avoid language that sounds like formal legal, medical, or regulated advice
- If the contributor mentions solutions, keep them broad and non-promotional
- Add a short bio if that supports credibility

### Markdown skeleton

```
## Expert advice: [Commentary title]

**By [Name], [Job title], [Organisation]**

[Paragraph 1: What stands out in the findings.]

[Paragraph 2: Why these findings matter in practice.]

[Paragraph 3: What organisations, leaders, or stakeholders may need to pay attention to.]

[Optional paragraph 4: Measured reflection on response, support, or next steps.]

### About [First name]

[Short bio covering expertise, role, and relevance.]
```

## 10. Health Shield final thoughts / conclusion

### Purpose

Close the report with a calm, credible summary that reinforces the main message and Health Shield’s perspective.

### When to use

Use near the end of every whitepaper.

### Required inputs

| Input                | Notes                         |
| -------------------- | ----------------------------- |
| Main report takeaway | The strongest closing message |
| Key supporting stats | Usually 2 to 3                |
| Brand perspective    | Supportive and relevant       |
| Named spokesperson   | Optional                      |
| Campaign connection  | Optional but helpful          |

### Writing guidance

- Summarise the challenge clearly
- Bring the report back to the reader’s reality
- Use Health Shield’s voice as a trusted guide
- Keep product mentions light
- Focus on support, understanding, and practical relevance
- Avoid a hard sales close
- Keep the final note steady and confident

### Markdown skeleton

```
## Health Shield final thoughts

### [Optional conclusion title]

**By [Name], [Job title], Health Shield**

[Paragraph 1: Restate the scale or significance of the issue using evidence from the report.]

[Paragraph 2: Explain why the findings matter for the intended audience.]

[Paragraph 3: Offer a measured Health Shield perspective focused on support, access, prevention, or wellbeing.]

[Paragraph 4: Close with a calm, clear final thought tied to the report theme.]
```

### When to use

| ----------------- | -------------------------------------- |
| Full report title | Final approved title                   |
| Organisation name | Usually Health Shield Friendly Society |
| Publication year  | Final year                             |
| Publication month | Optional but recommended               |

### Writing guidance

- Keep the citation wording simple
- Use the final approved report title exactly
- Make sure the organisation name is correct

```

**Health Shield Friendly Society. ([Year]). _[Full report title]._ Published [Month Year].**
```
